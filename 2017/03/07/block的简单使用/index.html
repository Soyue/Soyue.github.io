block的简单梳理

发表于 2017-03-07 | 分类于 iOS基础 | | 阅读次数

刚开始学习OC时，block是一个令我感到十分头疼的语法，尤其是看着它的写法，让我感到很是奇怪，后来使用多了，愈发感到它的便捷与强大，在开发中更是离不开block。就block的使用，我做一个简单的总结，就当自己对基础知识的一次梳理吧。

block的基础使用

在开发中我们一般使用bock进行传值，类似的功能实现还有代理和通知，不过相比较于代理和通知，block更加的方便。

block为什么使用copy修饰符

@property (nonatomic, copy) void(^myBlock)();

默认情况下，block是存档在栈中，可能被随时回收，需要copy操作使其在堆中保留一份, 相当于一直强引用着。这也就是我们在定义block的时候用的是copy。而不是weak或者其他。

block使用的注意点

前面说了block使用copy修饰，则self强引用着block，而此时如果block里面使用它所在的类的对象(即self)的话，那么这个对象将永远无法被释放，即为循环引用，从而造成内存泄漏。如：

解决办法，在代码前面加上下面这句就可以了。

__weak typeof (self) weakSelf = self;
